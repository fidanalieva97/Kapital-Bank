Layihə : Kapital Bankın mobil tətbiqi
Hazırlayan: Əliyeva Fidan
Məqsəd
Bu test planının məqsədi Kapital Bank sistemləri üçün test fəaliyyətlərinin həcmini, yanaşmasını, resurslarını və cədvəlini müəyyən etməkdir.
Test Ediləcək Modul və Funksiyalar
•	İstifadəçinin öz hesabına giriş etməsi
•	Hesab və kart əməliyyatları
•	Ödənişlər və köçürmələr
•	M10 ilə integrasiya
•	Cachback və ƏDVgerial bölmələri
•	Bildirişlər (SMS, Push)
Test Yanaşması
Testlər aşağıdakı mərhələlər üzrə aparılacaq:
•	Requirement analizi
•	Test case hazırlanması
•	Test data hazırlanması
•	Test icrası
•	Bugların qeydiyyatı və fix edilməsi
•	Regression testləri
Test Növləri
•	Manual testing
•	Functional testing
•	Smoke Testing
•	Sanity Testing
•	Regression Testing
•	Performance Testing
•	Təhlükəsizlik Testing
•	UAT
Test Environment
•	OS: İOS, Android
•	Mobil Cihazlar : İphone 15 Pro Max, Samsung S25 ultra
•	Network : Wi-Fi
•	Browser : Google Chrome, Microsoft Edge
Rollar və Məsuliyyətlər
•	QA Engineer – testlərin hazırlanması və icrası
•	QA Lead – test prosesinin idarə olunması
•	Developer – defectlərin aradan qaldırılması
•	Product Owner / Business Analyst – tələblər və prioritetlər

Giriş Kriteriyaları (Entry Criteria)
•	Tələblər təsdiqlənmiş olmalıdır
•	Test mühiti hazır olmalıdır
Çıxış Kriteriyaları 
•	Critical və High severity defectlərin bağlanması
•	Test coverage-in təsdiqlənməsi
•	UAT təsdiqinin alınması
